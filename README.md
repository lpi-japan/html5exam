## 現在進行中の改定プロジェクトの概要

### 背景と動機

現行試験（ver.2.5）は `objectives_level1.md` / `objectives_level2.md` で定義された1系統2レベルの構成になっている。しかし、旧 Level 1 は HTML/CSS 領域、旧 Level 2 は JavaScript・Web API 領域の内容であり、実態として「レベルの違い」ではなく「領域の違い」に近い状態になっていた。

この結果、プログラミング寄りの受験者がマークアップ領域の Level 1 を合格しなければ Level 2 に進めないという「学習順の不自由」が生じていた。

### 新試験の構成

この課題を解消するため、新試験では領域を **マークアップ** と **プログラミング** の2つに分割し、それぞれに Level 1 / Level 2 を設ける計4試験構成とする。

以下は旧試験との対応のおおよそのイメージ（厳密な1対1対応ではない）。

| 試験 | 出題範囲ファイル | 旧試験との対応イメージ |
|------|----------------|----------------------|
| Webプロフェッショナルマークアップ Level 1 | `objectives_markup.md`（1章） | 旧 Level 1 の HTML/CSS 系を再編 |
| Webプロフェッショナルマークアップ Level 2 | `objectives_markup.md`（2章） | ほぼ新設（旧 Level 2 は JS 中心のため） |
| Webプロフェッショナルプログラミング Level 1 | `objectives_programming.md`（1章） | 新設（旧試験に相当なし） |
| Webプロフェッショナルプログラミング Level 2 | `objectives_programming.md`（2章） | 旧 Level 2 の JS 基礎 ＋ 旧 Level 1 の Web API 概要を再編 |

マークアップ・プログラミングのそれぞれに独立した Level 1 の入口を設けることで、受験者が自分の専門領域から学習を始められる構成にする。

## 作業ルール

### 変更の種類とコミット分割

出題範囲への変更は以下の3種類に分類し、**種類をまたいで同一コミットにまとめない**。

| 種類 | 内容 | 備考 |
|------|------|------|
| **リファクタ** | 出題内容を増減させずに、記載をよりMECEで整理されたツリー構造に書き換える | 内容の実質的な変更なし |
| **試験間移動** | 出題項目を試験をまたいで移動させる（例：マークアップ→プログラミング） | 全体の出題量は変わらない |
| **内容増減** | 出題項目を追加・削除する | 必ず客観的な根拠が必要（後述） |

### 内容増減の判断基準

出題項目を増減させる場合は、必ず客観的な理由を明示し、`decision_records.md` の判断記録と連動させる。どういう判断に基づいて増減させたかをいつでも説明できる状態を維持すること。

理由なく「なんとなく追加・削除」しない。増減を伴う変更は慎重に、かつレビュー可能な形でコミットする。

### 出題範囲文の書式

出題範囲文は以下の階層構造で記述する。

```
試験 (Exam)       … Markdown 見出しレベル 1（ファイル内に Level 1 / Level 2 の2試験）
  主題 (Section)  … Markdown 見出しレベル 2
    副題 (Topic)  … Markdown 見出しレベル 3、末尾に (重要度: N) を付記
      概要        … 単文または数個の箇条書き。その副題で問いたい視点を表す
      詳細        … 2階層の箇条書き
```

**表記ルール**

- アルファベット単語の前後には半角スペースを入れる（例: `HTML 要素`）
- カッコは半角 `()` を使い、前後に半角スペースを入れる
- コマンド・関数・プロパティ名などはバッククォートで囲む（例: `` `display` ``）
- 技術仕様への参照は footnote 形式（`[^label]` を本文に、`[^label]: 仕様名 — URL` をファイル末尾に）で記載する。仕様名・レベルは必ず MDN や W3C の公式ページで確認してから記入する。記憶や推測で書かない。確認できない場合は `（要出典）` と記す。

## 現在の状況と当面の課題

### 作業優先度

検討内容は大きく2つある。

- **Level 1 の整備**: 現行試験の内容を再編する形で詳細を詰めやすいため、先行して進める
- **Level 2 のスコープ検討**: 上位試験として上級者のスキルを確認するための新規要素の追加が必要になる可能性があり、Level 1 の整備と並行して検討する
